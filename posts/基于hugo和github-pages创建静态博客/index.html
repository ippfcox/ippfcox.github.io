基于Hugo和Github Pages创建静态博客

May 10, 2023

创建博客的过程

  1. 在自己的github创建两个仓库，一个是ippfcox.github.io，用于github pages，另一个是blog，用于保存博客源码，这个仓库设置为私有了
git@github.com:ippfcox/ippfcox.github.io.git
git@github.com:ippfcox/blog.git
  1. 创建网站
hugo new site blog

创建一个blog目录，并初始化

  1. 配置远端地址
cd blog
git init
git remote add origin git@github.com:ippfcox/blog.git

配置远端地址为刚刚初始化的仓库，配置了这个才能通过git submodule配置主题

  1. 配置主题，我使用PaperMod
git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod
echo "theme = 'PaperMod'" >> config.toml
  1. 此时可以预览
hugo server
  1. 配置.gitignore，修改默认分支，推送一个版本
.hugo_build.lock
.vscode

git add *
git commit -m "initial commit"
git branch -m master main
git push origin main
  1. 创建一篇文档
hugo new posts/基于Hugo和Github-Pages创建静态博客.md

注意修改draft为false，修改config中的url和名称等

  1. 发布
hugo

生成静态文件到public中，在public目录初始化git仓库并添加远端地址

cd public
git init
git remote add origin git@github.com:ippfcox/ippfcox.github.io.git
git add *
git commit -m "initial commit"
git branch -m master main
git push origin main

稍等一下，就可以在ippfcox.github.io上看到了

  1. 总结
  • 有些主题还需要npm install，我嫌麻烦就不用了
  • 发布文件与源文件最好分成两个仓库，源文件仓库（blog）配置成私有挺ok的
